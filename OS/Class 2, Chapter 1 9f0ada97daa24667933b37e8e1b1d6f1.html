1.0 Introduction

The importance of an Operating System lies in its ability to create abstractions and allow programs to operate on it, without concerning themselves with the underlying hardware and schematics of the specific architecture.

Another job for the OS to be an interface for a User to interact and operate on processes and programs.

Key things to be aware of

  • What is a Kernel?
    • It is a special program that provides services to running programs. Each running program, is called a process.
    • The kernel sits between the hardware and all other processes, acting as an intermediary.
    • One of its main functions is to abstract away the physical differences between individual computers and hardware systems.
    • Other key functions of the kernel is to provide security and stability. It implements a sense of Protected Memory Space.

Operating System Interpreter.

Software Bifurcation (bifurcation means the division of something into two branches or parts)

  • Kernel Space (OS’s heart)
  • User Space (User’s Mind)
  • Kernel is a program just like user program but with much more software and hardware privileges.
  • User space is restricted so if a user program wishes to do something beyond its privilege, it has to request the kernel.
    • Such user requests are called system calls (Like we did in CAAL).

“Hart” in the book or course is a reference to a core. Hardware thread.

The structure that we learned in CAAL with the PC (Program Counter), IM (Instruction Memory), RF (Register File), ALU (Arithmetic and Logic Unit) and DM (Data Memory) is basically one core itself. Single core.

This is similar to the concept of Multiprocessor systems, but instead of multiple processors we instead have multiple cores in one microprocessor.

The way that these cores interact with each other is based on their Scheduling (The processes are scheduled based on weights or any factor determined) . In XV6 the scheduling is based on ROUND ROBIN.

1.1 Processes and memory

  • What are processes?
    • Processes are currently running programs.
    • Each process has memory containing instructions, data and a stack (The stack used for function calling / procedure calling).
    • A computer typically has many processes but only a single kernel.
    • An xv6 process consists of user-space memory (instructions, data, and stack) and per-process state private to the kernel.
    • The kernel associates a process identifier or PID , with each process.
    • A process may create a new process using the fork system call. Fork gives the new process an exact copy of the calling process’s memory, both instruction and data. In the original process, fork returns the new process’s PID. In the new process, fork returns zero. The original and new processes are often called the parent and child.
