Attributes:

  • Simple attributes - single component with single existence
    • Call number, name, etc.
  • Composite attributes - multiple components, split into separate existance.
    • e.g. address can be split into street, number, zip, city, country. etc.
  • Single-valued attributes
  • Multi-valued attributes
  • Derived-attributes: Attributes that are dervied from other attributes e.g. Age from data of birth.

Keys:

  • Candidate key : Set of attributes that unqiuely identifies each record.
  • Primary key ; Candidate key selected for unqiue identification.
  • Composite key : Combination of multiple attributes to create a candidate key.

Foreign keys are important for creating relational databases.

Natural keys are keys that occur natually and are inherently unique, such as country names.

Surrogate keys are keys that may or may not be originally natural and are adjusted to satify requirements e.g. Country Initials . Surrogate keys are also custom keys that we create ourselves e.g Student ERP.

These things are decided by the company’s requirements and policy.
