Roadmap

feature extraction: Extracting the specific attributes we need from within the data that will actually be used by the machine learning model.

Some of the dataset is used as test date for evaluating the final model. NEVER use the training data as the test data, because you will not be sure whether or not the model has been overfitted .

Feature scaling: It is a process where we scale the values within the features to a normalized or scaled amount, such that features which are inherently higher in value to others (e.g. age to salary), does not have a significant influence in the training of the machine learning model.

Dimensionality reduction: It is a method for transforming data from a high-dimensional space into a low dimensional space. The goal is to retain some meaning full properties of the original data such as its intrinsic (essential) dimension, variance or structure. (Simplier explanation) Dimensionality reduction is the process of reducing the number of features (or dimensions) in a dataset while retaining as much information as possible. This may be done to reduce the complexity of a model, improve performance or make data easier to visualize.

Think of dimensionals like the power of the variables in quadratic formulas.

Missing data handling : The easiest way to handle missing data is to remove the features it belongs to.

CRISP-ML(Q)

CRoss Industry Standard Process for the development of Machine Learning applications with Quality assurance methodology.

It integrates quality management principles into the machine learning lifecycle, making it a structured approach for building, deploying, and maintaining machine learning models.

The main stages are:

  1. Business and Data Understanding
  1. Data Engineering (Data Preparation)
  1. Machine Learning Model Engineering
  1. Quality Assurance for Machine Learning Applications
  1. Deployment
  1. Monitoring and Maintenance

Dropping data isnt always a good solution, so its best that we have alternative ways, such as imputing missing values (represent (something, especially something undesirable) as being done or possessed by someone; attribute).

An easy way to impute the missing data is to replace it with the mean of the values of the entire feature column.

We can do this using the simpleImputer class from scikit-learn.

Encoding

How to handle categorial data?

Many machine learning models, particularly those based on mathematical operations, require input features to be numerical. Categorical data (like colors) must be convereted to numerical representations. For this we need encoding techniques that achieve this.

One hot encoding

The idea behind one hot encoding is that we create a new dummay feature for each unqiue value in the nominal feature column.

To achieve this we can use the pandas method 

df.get_dummies();

There are also other types of encodings like ordinal or label.

Ordinal Encoder, creates an ordering based on any specification you want.

The difference between ordinal and label is that labal will convert to a in64 series, while ordinal will convert to float64

Scaling

Feature scaling

Methods for feature scaling: Min-Max (Normalization) and Z-score (Standardization)

Normalization and standardization have their own uses depending on the context of the data.

  • When using distance-based algorithms (like kNN or clustering
    algorithms), normalization is preferable.
  • Normalization is more intuitive and offer better interpretability than
    standardization.
  • Standardization is less sensitive to outliers and, hence, is preferable
    when outliers are present.
  • Standardization is required/preferable when working with Neural
    Networks or PCA
